## What is RAG?

RAG (Retrieval-Augmented Generation) combines large language models with external knowledge bases to provide grounded, accurate responses.

## Why It Matters

RAG reduces AI hallucinations by 80%, enables LLMs to access up-to-date information, and provides verifiable citations.

## How to Implement

1. Prepare knowledge base documents
2. Deploy vector database (Pinecone, Weaviate)
3. Implement retrieval pipeline
4. Integrate with LLM API
5. Evaluate and iterate

## Frequently Asked Questions

**What is the difference between RAG and fine-tuning?**
RAG adds knowledge at inference time; fine-tuning updates model behavior.

**What vector databases support RAG?**
Pinecone, Weaviate, Milvus, and pgvector are popular choices.
